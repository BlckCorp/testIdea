# testIdea

Репозиторий для идей, которые мы придумываем, проверяем и превращаем в MVP.

Главная логика репозитория: **каждая MVP-идея должна иметь отдельную полноценную демо-страницу**, которую можно открыть и показать как пример будущего продукта.

## Как смотреть

- `index.html` — каталог всех идей.
- `mvp.html?id=...` — описание MVP: суть, аудитория, состав, цена, тестирование.
- `visual.html?id=...` — общий визуальный мокап идеи.
- `pages/<id>.html` — полноценная отдельная демо-страница MVP.

## Отдельные демо-страницы

```text
pages/love-site.html
pages/love-archive.html
pages/date-box.html
pages/future-letters.html
pages/couple-passport.html
pages/love-order.html
pages/avito-pack.html
pages/master-landing.html
pages/electrician-audit.html
pages/document-helper.html
```

## Что значит “полноценная демоверсия”

Не просто описание и не просто мокап. Демо-страница должна показывать, как MVP будет выглядеть для клиента:

1. отдельный URL;
2. законченный экран/страница продукта;
3. пример содержимого;
4. визуальная структура;
5. кнопки или интерактив, если это уместно;
6. блок “что получает клиент”;
7. блок “что нужно от клиента”.

## Как добавить новую идею

1. Добавить идею в `data/ideas.js`.
2. Создать отдельный файл `pages/<id>.html`.
3. В каталоге кнопка “Страница” должна вести на эту демо-страницу.
4. В `mvp.html?id=<id>` должна быть ссылка на отдельную страницу MVP.
