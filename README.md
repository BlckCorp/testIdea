# testIdea

Репозиторий для идей, которые мы придумываем, проверяем и превращаем в MVP.

Теперь это не только база заметок, но и статическая витрина MVP: стартовый `index.html` показывает идеи как каталог с превью, статусами, ценами и кнопкой просмотра.

## Быстрый просмотр

- `index.html` — стартовое меню всех MVP.
- `mvp.html?id=love-site` — универсальная страница конкретной идеи.
- `data/ideas.js` — единая база идей для каталога и страниц.
- `assets/style.css` — общий визуальный стиль.

## Зачем это репо

Здесь храним не просто случайные мысли, а заготовки под реальные продукты и услуги:

- подарки для пар;
- сайты-приглашения;
- услуги для Авито;
- идеи для малого бизнеса;
- ИИ-инструменты;
- электрика и бытовые услуги;
- документы, лендинги, упаковка объявлений.

## Как устроено

```text
index.html     — стартовая витрина MVP
mvp.html       — универсальная страница просмотра идеи
data/ideas.js  — расширяемый список идей
assets/        — стили и будущие скриншоты/картинки
ideas/         — оформленные идеи и продуктовые линейки
templates/     — шаблоны карточек, офферов и объявлений
backlog/       — сырые идеи, которые ещё надо разобрать
```

## Как добавить новую идею

1. Открыть `data/ideas.js`.
2. Скопировать любой объект идеи.
3. Поменять `id`, `title`, `category`, `price`, `tagline`, списки и CTA.
4. В `index.html` карточка появится автоматически.
5. Страница будет доступна по адресу:

```text
mvp.html?id=your-new-id
```

## Правило репозитория

Каждая идея должна отвечать на три вопроса:

1. Что продаём?
2. Кому это нужно?
3. Как сделать MVP за 1 вечер?

## Быстрый приоритет

Сначала развиваем идеи, которые можно быстро проверить через Авито, ВК, личные чаты или простой лендинг.

Текущий первый фокус: `Сайт вашей любви` как демо-продукт для ниши подарков для пар.
